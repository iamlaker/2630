# 03 - 发现、确认和复用输入规则

**What to build:** 建立规则发现、候选生成、不可变版本、确认编辑和模板复用的后端能力，为独立的规则集确认与维护工作台提供稳定服务边界；后续计算只使用满足状态要求的规则。

**Blocked by:** 02 - 导入模板并生成指标目录

**Status:** ready-for-human

- [x] 系统能够为每个输入指标生成源单元格候选、公式链、置信度和规则版本。
- [x] 无法唯一追踪源单元格的指标不会被自动写入，并显示待确认状态。
- [x] 管理员能够确认、编辑或拒绝候选规则，并看到变更前后的规则信息。
- [x] 规则包含显示项目、年度映射、单位、调整模式、步长、范围、五年联动策略和模板指纹。
- [x] 同一指标在兼容模板版本中能够复用已确认规则；不兼容时会进入待确认状态。
- [x] 规则确认和编辑被记录到审计日志。

## Comments

- 实现摘要：新增独立 `input_rules.py`，提供公式依赖图、候选规则发现、SQLite 不可变规则版本、查询筛选、确认、手工编辑、拒绝、兼容复用、变化检测和完整审计接口；扩展统一 `WorkbookEngine` 的单元格公式/值读取边界，并让正向测算直接消费已确认规则记录。待确认、拒绝、变化和不支持规则返回 `pending_rule_confirmation`，不会回退写入汇总展示表。
- 规则发现策略：从《汇总展示表》每个输入指标的 2026-2030 显示单元格出发，解析直接、跨表和多级 A1 引用；逐路径记录工作表、地址、公式、叶子类型、最大深度、循环和无法解析状态。仅将公式链终点的常量单元格列为候选；唯一且五年完整、无诊断异常时给出高置信度，但仍保持人工确认。`INDIRECT`/`OFFSET` 等动态引用明确标记为不支持。
- 测试命令和结果：`$env:PYTHONPATH='.'; python -m unittest tests.test_input_rules tests.test_forward_calculation tests.test_template_import -v`，任务 1-3 共 20 项全部通过；覆盖唯一/跨表/多级追踪、多候选、循环、无法解析、候选置信度、确认编辑拒绝版本历史、规则复用与变化、计算阻止/消费和审计。
- Excel COM 验证：已使用 `ExcelComWorkbookEngine` 隔离副本实际追踪《汇总展示表》第 107 行 2026 年公式，找到唯一底层候选 `C19`，无循环和解析错误；任务 1 真实正向测算及任务 2 真实模板目录测试也在全套测试中通过。原模板 SHA-256 前后均为 `a7c61eda5cbfbff06770d74651674196dcd2e90d519341f78b200eee6b8c61aa`。
- 已知限制：当前轻量公式解析器支持常见 A1 单元格引用，不展开命名区域、结构化引用、外部工作簿、区域引用中的每个单元格及动态引用；这些情况保持待确认或不支持。调整方式可依据分组/单位生成初始建议，但范围、规模步长及无法可靠推断的配置仍明确标记 `configuration_pending`，需管理员确认。COM 中文名称乱码为既有环境限制，规则仍保存工作表标识与地址。
- 下一步建议：可以进入任务 4；前端或 API 层直接调用 `RuleService.list_rules`、`confirm_rule`、`edit_rule`、`reject_rule`，计算入口使用确认后的规则记录。后续若模板大量使用命名区域或复杂 Excel 公式，再按实际样本扩展解析器。
- 扩大测试补充（2026-07-16）：对真实模板目录中的 82 个输入指标、2026-2030 五个年度执行只读公式图扫描。54 个指标最大链深为 2；其余包含 3-41 层依赖，14 个指标出现多候选，15 个指标至少一年没有可用候选；复杂 RWA 等加工链单年可展开至数百个叶子候选。新增测试覆盖嵌套 `IFERROR`/`ROUND`、乘除与加减加工、多叶子比例公式、9 层深链，以及引用工作表/单元格读取失败时不中断全量发现。
- 扩大测试后的策略调整：新增 `processed_formula` 和 `read_error` 诊断。加工公式即使最终只有一个常量叶子，也只给低置信度并保持 `pending_confirmation`；引用读取失败作为明确链节点保存，不再使整批扫描失败。动态引用、超深链和数百候选仍不尝试猜测源单元格。
- 任务维护说明：管理员可视化确认、调整配置维护、规则集完整性校验和激活发布已拆为任务 04；本任务保留已完成的规则发现与版本服务基础。
