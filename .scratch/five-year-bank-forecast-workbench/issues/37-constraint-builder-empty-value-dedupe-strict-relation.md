# 37 — P0：约束构建器空值拦截、同组查重与严格不等号忠实显示

**What to build:** 修三个约束构建器缺陷：
1. **空值静默变 0**：`addConstraintGroup`（web/app.js:898-946）中 `Number(el.value)` 对空 number 输入得 0 且通过 `isFinite` 校验（app.js:907、918-920），"请输入约束目标值"的 alert 永远不会触发。改为：空字符串先拦截，标红该输入并提示，不添加记录。
2. **同组查重**：同一指标 + 同一关系 + 同一年份范围可重复添加成多个组（对比 `addReverseVariable` 有查重，app.js:1099）。添加时若已存在相同 指标+关系+年份覆盖 的启用组，alert 提示并拒绝（或提供"替换"选项，取实现简单者：拒绝并提示）。
3. **严格不等号忠实显示**：构建器选了 `>` 或 `<` 后，单记录约束卡的关系下拉只有 ≥/≤/=（app.js:1418），显示为 ≥/≤ 造成信息丢失。单记录卡的关系下拉补上 `>`/`<`，与 `group_label`/`relation` 元数据保持一致（后端 kind 映射不变：>`≥`→min、`<`≤`→max）。
